# asdm-context-build 操作指令

## 目的
本指令指导 AI 模型使用 Context Builder 工具集为当前工作区生成上下文。

## 语言检测

在生成上下文文件之前，您必须检测并使用当前环境的响应语言：

1. **检测响应语言**：分析环境设置以确定主要语言：
   - 检查系统/用户语言设置或环境配置
   - 识别项目文档和注释中使用的主要语言
   - 根据工作区上下文确定语言偏好

2. **应用语言一致性**：确保所有生成的上下文文件使用检测到的语言：
   - 所有 markdown 文件、注释和文档使用相同的语言
   - 在所有生成的上下文文件中保持语言一致性
   - 遵循检测到的语言的写作规范和格式

3. **支持的语言**：
   - 英语 (en)
   - 中文 (zh)
   - 根据环境检测需要支持其他语言

**重要提示**：语言检测是任何上下文生成之前的**第一步**。所有输出在整个过程中必须一致地使用检测到的语言。

## 生成上下文的步骤

### 1. 初始化上下文目录
如果 `.asdm/contexts/project_workspace/` 目录不存在，请在工作区根目录创建它。

### 2. 分析当前工作区

1. 读取已经生成的所有模块上下文 `.asdm/contexts/`。
2. 分析当前工作区以了解：
- 技术栈和框架
- 项目结构和组织
- 现有代码模式和约定
- 部署配置和环境

### 3. 使用规范
使用 `spec/` 目录中的规范作为起点，但根据实际工作区分析进行定制。
**重要提示**：将所有规范内容翻译并适配到检测到的语言，确保所有文本、示例和解释的自然和准确的本地化。

### 4. 生成上下文文件 
**确保生成的文件使用语言检测步骤中检测到的语言：**

1. 根据所有模块上下文和当前工作区代码，按需生成上下文文件
**每个后续文件必须使用相同的检测语言：**
   - **standard-project-structure.md** - 标准项目结构
   - **standard-coding-style.md** - 标准编码风格
   - **data-models.md** - 数据模型和关系
   - **deployment.md** - 部署配置
   - **api.md** - API 定义和文档
   - **architecture.md** - 架构概述

**每个额外上下文文件的处理流程**：
- 用户指定要生成的文件
- 使用检测到的语言单独生成请求的文件
- 与 index.md 和之前生成的文件保持一致
- 在继续下一个文件之前等待用户审核

这种分阶段方法确保：
- 高效的 token 使用
- 每个文件的高质量输出
- 用户控制生成顺序
- 能够根据反馈进行调整

2. 根据已生成内容和加载的上下文，生成工作区的索引上下文**index.md** - 工作区索引和指南
   - 提供整个工作区的概述
   - 包含指向其他上下文文件的导航链接（后续生成）
   - 总结工作区结构和关键组件

**重要提示**：此步骤仅生成 index.md，以避免过多的 token 使用并保持高质量生成。

### 5. 包含 Mermaid 图表
在适用的情况下，包含 mermaid 图表用于：
- 实体关系图
- 类图
- 时序图
- 部署图
- 架构图

### 8. 链接到源代码
包含相关源代码文件的链接以便详细参考。

## 使用方法
要使用此指令，只需按照上述步骤操作。AI 模型应分析工作区并生成全面的上下文文件，这些文件可通过上下文注入供其他工具集使用。
